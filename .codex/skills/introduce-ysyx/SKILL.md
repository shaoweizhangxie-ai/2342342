# 介绍一生一芯

## 工作流程

1. 读取 `references/project-introduction.md`，将它作为项目长期信息的主要依据。
2. 根据用户场景调整篇幅和表达，但不改变关键事实：项目性质、发起背景、核心愿景、贯通 CS 与 EE 的培养方式，以及流片机会的限制。
3. 若用户询问当期报名时间、人数、课程任务、考核规则、认证等级或流片条件，先查阅一生一芯官网最新页面；这些信息可能变化，不凭记忆作答。
4. 明确区分“处理器设计”“SoC 集成”“物理设计”“流片”和“回片测试”，避免将完成课程等同于必然获得流片机会。
5. 默认使用中文回答；用户使用英文或明确要求英文时，使用项目官方英文名 `One Student One Chip (YSYX)`。

## 输出方式

- 用户只问“什么是一生一芯”时，先给出 100～200 字概览，再按需要补充。
- 用于宣讲或 README 时，按“背景—理念—路径—收获—参与方式”组织内容。
- 面向初学者时解释 `RISC-V`、`SoC`、`EDA` 和“流片”等术语，不堆砌缩写。
- 面向技术读者时突出软硬件协同、处理器前端、系统软件、验证和后端流程。
- 用户询问是否适合参加时，同时说明开放性与学习强度，避免只做宣传性描述。
- 若输出包含时效性事实，附官方页面链接并标注查询日期。

## 准确性边界

- 不声称所有参与者都能完成芯片制造或免费流片。
- 不把“一生一芯”描述为单纯的 Verilog、Chisel 或 RISC-V 入门课。
- 不自行推断当期招生、收费、证书或选拔政策。
- 对未在参考资料或官网确认的数据使用审慎表述，不编造规模、成果或合作方。
